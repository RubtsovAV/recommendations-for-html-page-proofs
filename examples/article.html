Качественная вёрстка это:

  • Читабельные и понятные названия классов
  • Возможность переиспользования повторяющихся блоков (например, использовать одинаковую вёрстку для верхнего, нижнего и бокового меню)
  • Использование отступов в 4 пробела (вместо таба)
  • Использовать эту рыбу для проверки отображения содержимого текста, редактируемого пользователем
  • Не останавливаться на достигнутом...

Основные требования:

  1. Блок <head> одинаковый на всех страницах
  2. CSS вынесены в файлы и подключаются через <link> и только внутри <head>
  3. Атрибуты style="..." и id="..." недопустимы
  4. Весь JavaScript в отдельных файлах и подключается перед </body>
  5. Перечень подключаемых файлов (стили, скрипты, шрифты) должен быть одинаковым на всех страницах
  6. Все файлы лежат в соотвествующей папке js, css, images. Ссылки на внешние ресурсы недопустимы.
  7. Шапка и подвал страницы внутри тега <header> и <footer>
  8. Подвал всегда прижат к низу страницы, даже если на ней мало текста
  9. Боковые sidebar-ы (колонки) должны быть внутри <aside>
  10. Заголовки h1-h6 допускаются только там, где без них нельзя обойтись. Например, внутри <article>.
  11. Список товаров, статей или новостей должен быть внутри тега <main> с тегом <h1> внутри
  12. Содержание новости или статьи должно быть внутри тега <article> с тегом <h1> внутри
  13. В блоках предполагающих редактирование через wisywig редактор не должно быть атрибута class="..."
  14. Тег <h1> всегда без атрибутов (class="..." и др. недопускаются)
  15. Валидность кода https://validator.w3.org/

Теги фоматирования, которые могут содержаться в пользовательском тексте:

  • <strong>
  • <b>
  • <i>
  • <em>
  • <u>
  • <ins>
  • <s>
  • <del>
